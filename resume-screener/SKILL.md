# Resume Screener

## 核心架构

```
JD + 简历 → [特征提取] → [JD类型判定] → [硬门槛过滤] → [加权评分] → [结构化呈现]
```

---

## 第0步：JD类型判定

**技术岗判定条件（任一满足即判断为技术岗）：**
- 包含编程语言：`Python / Java / Go / C++ / Rust / JavaScript / TypeScript / SQL`
- 包含框架/平台：`Spring / Django / React / Vue / TensorFlow / PyTorch / Docker / Kubernetes`
- 包含技术领域：`后端 / 前端 / 算法 / 数据开发 / 测试开发 / 运维 / 全栈 / 数据挖掘 / 机器学习 / AI`
- 包含中间件/数据库：`MySQL / Redis / MongoDB / PostgreSQL / Kafka / Elasticsearch`
- 含明确技术栈描述（如"熟悉Linux"、"掌握分布式系统"）

**管培/非技术岗判定条件（任一满足即判断为管培岗）：**
- JD含：`管培生 / 商务 / 运营 / 市场 / 人力 / 销售 / 行政 / 客服 / 产品 / 设计 / 财务 / 法务 / 采购 / 物流 / 供应链`
- JD**不包含**任何技术岗关键词
- JD偏重软性技能：沟通能力 / 团队协作 / 责任心 / 执行力 / 抗压能力

**判定优先级：** 技术关键词优先 → 无技术关键词按非技术 → 均无默认管培权重

> ⚠️ 每次输出必须明确写出「岗位类型判定结果及判定依据」

---

## 第1步：特征提取

从 JD 提取：岗位名称、职责、硬性要求（学历/毕业时间/专业/年限）、技术栈（仅技术岗）、软性要求、加分项

从简历提取：毕业时间、学校、学历、专业、GPA、所在地、实习经历（企业/时长/职责/成果）、项目经历、竞赛获奖、证书、品格信号

---

## 第2步：硬门槛过滤

**一票否决项（不通过直接筛除）：**
1. **毕业时间不匹配**（JD明确要求某届，不匹配则否。JD未明确的跳过此项）
2. **学历不达标**（JD要求本科，大专直接否）
3. **专业零交集+技能栈为零**（JD招后端，简历无关且无编程经验）
4. **非在读**（实习岗要求在校生，已毕业直接否）

> "稍微搭边可以先过"→标记为「待定」进入评分池

**硬门槛结果表：**

| 候选人 | 毕业时间 | 学历 | 专业交集 | 在读状态 | 结论 |
|--------|---------|------|---------|---------|------|
| 姓名 | ✅/❌/🔄无要求 | ✅/❌ | ✅完全对口/▲部分相关/❌无关 | ✅/❌ | ✅通过/🚫淘汰/▲待定 |

---

## 第3步：加权评分（双模式）

### 模式A：技术岗权重（修订版）

| 维度 | 权重 | 评分方法 | 说明 |
|------|:----:|---------|------|
| 学历 | **30%** | 动态分档表查表 | 按 JD 学历要求查对应分档表 |
| 项目经历 | **30%** | 专家打分 0-100% | 项目复杂度、规模、技术深度、是否企业实战 |
| 技术栈 | **15%** | JD技术栈覆盖率 0-100% | 已体现在项目中的不重复扣分 |
| 专业 | **10%** | 匹配度 0-100% | 优先专业=100%，相关=80%，无关=30-60% |
| 竞赛 | **10%** | 含金量 0-100% | ACM/数模/国赛=100%，省级减半，校级再减半 |
| 品格 | **5%** | 信号累计 0-100% | 项目组长/班长→领导力，社团→协作，励志奖学金→刻苦 |

### 模式B：管培/非技术岗权重（修订版）

| 维度 | 权重 | 评分方法 | 说明 |
|------|:----:|---------|------|
| 学历 | **30%** | 动态分档表查表+专升本附加分 | 管培看潜力>学校，专升本在总分上加+1分 |
| 实习/项目 | **20%** | 专家打分 0-100% | 完整度、复杂度、与企业需求匹配度 |
| 竞赛/荣誉 | **10%** | 含金量与密度 0-100% | 国家级竞赛/奖学金>省级>校级 |
| 技能 | **15%** | 证书与工具覆盖 0-100% | 办公软件、语言能力、专业证书等实用技能 |
| 品格 | **10%** | 信号累计 0-100% | 领导力、沟通力、抗压力、自驱力 |
| 专业 | **15%** | 匹配度 0-100% | 优先专业=100%，相关=80%，无关=30-60% |

### 评分公式

```
单维度加权分 = 原始百分比(%) × 该维度权重(%)
总分 = Σ(所有维度加权分)
示例：学历原始值48%，权重30% → 学历加权分 = 48% × 30% = 14.4分
```

> ⚠️ 每个维度的原始百分比必须写明依据

## 可复现性保障

1. **步骤不可跳步**：严格按 特征提取→JD类型判定→硬门槛过滤→加权评分→结构化呈现 顺序执行
2. **每维度注明依据**：原始%取值必须附带明确理由
3. **分数公式公开**：每项加权分写出 `原始% × 权重 = 结果` 的算式
4. **保留硬门槛结果**：淘汰原因保留到最终输出
5. **排名表含所有维度列**：方便追溯分数来源
6. **同一JD+同一组简历→同一组分数**

## 注意事项

- **区分度原则**：分数覆盖 0-100 全区间
- **项目经历和技术栈有重叠**：评项目时已体现的技术栈不重复扣技术栈分
- **品格推测注明依据**
- **JD类型判定必须明确输出**
- **多岗位时每个岗位独立筛选**
