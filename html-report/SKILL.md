# HTML 报告生成模板

## 适用场景

简历筛选报告（含候选人排名、6维度评分、评分依据、行动建议+沟通策略）

## 设计规范

背景色 `#f5f7fa`，头部 `linear-gradient(135deg, #1a1a2e, #16213e, #0f3460)`，卡片白圆角 `border-radius: 12px`，字体 `Inter, PingFang SC, Microsoft YaHei`，标签绿 `#059669`/黄 `#d97706`/红 `#dc2626`，进度条细圆角 `height: 4px`。进度条颜色：≥70%绿 / 50-69%黄 / <50%红。

## 排版结构
1. 渐变色头部 — 标题 + 元信息网格
2. 筛选总览 — 三色汇总卡片
3. 排名表格 — 完整维度列
4. 候选人详情 — 每候选人一张卡片：6维度进度条 + 评分依据列表
5. 行动建议 — 必须按下方格式

## 行动建议格式（必须严格遵循）

### ✅ 推荐面试（≥70分）
每人写两个部分：候选人背景 + 🎯 沟通方向（具体建议面试问什么）

### 📌 待沟通确认（60-69分）
每人必须写两个部分：
- ✅ 优势（为什么值得联系）
- ❓ 需确认问题（具体该问什么）

### ❌ 不推荐（<60分）
一句话列出所有不推荐候选人，缺人可以指明优先联系对象。

## 关键实现
- 加权分 = pct × weight / 100
- 百分比提取用 `re.match(r'(\D+?)(\d+)%', ...)`
- 按总分从大到小排序
- 所有候选人必须有评分依据列表

## 常见陷阱
- 字符串分割提取百分比 → dict key 错误全显示0
- 只给高分写评分依据
- 行动建议缺少沟通方向和需确认问题
